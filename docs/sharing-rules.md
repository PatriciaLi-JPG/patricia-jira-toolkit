# Sharing Rules

Before opening a PR or sharing this repo:

- No Jira tokens, cookies, passwords, or personal API keys.
- No customer-sensitive ticket exports.
- No personal local paths such as `C:\Users\<name>\...` in shared instructions.
- No private Google Sheet IDs unless the team has approved shared access.
- Use `*.example.json` for shared defaults.
- Use `*.local.json` for personal config and keep it ignored by Git.

Recommended rollout:

1. Keep the repo private.
2. Start with `release-story-points-report` and `t3-list-followup`.
3. Have 1-2 Scrum Masters try the skills for one sprint.
4. Move team-specific Jira field IDs and team aliases into shared config only after validation.
